A user plugin for [**Equicord**](https://github.com/Equicord/Equicord) (a fork of
[**Vencord**](https://github.com/Vendicated/Vencord)).

> Simulates moderation actions (ban, kick, timeout, warn, mute, deafen) **100% locally**.
> Nothing is sent to Discord, no one is actually affected, and only **you** see the result.

---

## ⚠️ Important — read before using

This plugin is **purely visual and local (client-side)**. It has **no** real moderation power.
Specifically:

- **No real action happens.** No user is banned, kicked, muted, or warned.
- **Nothing is sent to Discord.** There are no requests to moderation APIs, no endpoints called,
  and no network traffic generated by the actions.
- **The effect only exists on your screen.** The toasts, notifications, and history are rendered
  only on your client. No one but you sees anything.
- **No permissions are modified.** The plugin never changes roles, permissions, or member state —
  not yours, and not anyone else's.

## What the plugin does

When you right-click a user, the plugin injects entries that look **identical to Discord's native
ones** — no submenu, no custom icon. They appear in the same area as the real moderation actions
(right after **Roles** / **Open in Mod View**).

There are two formats, mirroring real Discord:

**Moderation actions** — **red** items (`Action username`) that open a modal:

| Action | Emoji | Duration? | Appearance |
| --- | --- | --- | --- |
| Timeout | ⏳ | Yes | Red item `Timeout username` |
| Kick | 👢 | No | Red item `Kick username` |
| Ban | 🔨 | No | Red item `Ban username` |
| Warn | ⚠️ | No | Red item `Warn username` |

**Voice toggles** — **checkboxes** (like the native Mute/Deafen), toggled instantly:

| Action | Emoji | State | Appearance |
| --- | --- | --- | --- |
| Mute | 🔇 | On/off | Red checkbox `Mute` |
| Deafen | 🙉 | On/off | Red checkbox `Deafen` |

Behavior:

- **Moderation actions:** open a **modal** with a **reason** (and a **duration**, for Timeout),
  log to the **history**, and show a **toast** + **colored notification**.
- **Voice toggles:** flip a **per-user local state** instantly. Checking logs to the history and
  notifies; unchecking shows a toast. The state is saved and the checkmark persists when you reopen
  the menu — but it is, of course, completely fake.

## How it works (technical overview)

The entire flow is strictly local:
`context menu → modal/toggle → local store → notification`.
At no point is there a `fetch`, `RestAPI`, or any call to moderation endpoints.

### File structure

```
Fake Moderator/
├── index.tsx                 # Plugin + menu patch (red items + checkboxes)
├── settings.tsx              # Settings (toggles, color, history, voice state, panel)
├── store.ts                  # History + voice-state store (persisted in settings)
├── utils.ts                  # Toast/notification, formatting, JSON export
├── constants.ts              # Action table, menu order, duration presets
├── types.ts                  # Strong typing for every structure
└── components/
    ├── ActionModal.tsx       # Reason + duration modal (moderation actions)
    └── SettingsPanel.tsx     # Panel: color, export, clear, history table
```

### Role of each module

- **`types.ts`** — Defines `FakeActionType` (`ban | kick | timeout | mute | deafen | warn`),
  `FakeActionRecord` (target, reason, duration, timestamp), `FakeVoiceState` (per-user mute/deafen),
  and `FakeActionMeta` (includes `kind: "moderation" | "voice"`). Strong typing throughout.

- **`constants.ts`** — The `ACTIONS` table is the **single source of truth**: the menu, the modal,
  and the settings are all derived from it. `MENU_ORDER` sets the order of the red items
  (Timeout → Kick → Ban → Warn) and `VOICE_ORDER` the order of the checkboxes (Mute → Deafen). It
  also defines the duration presets and the history cap (`MAX_HISTORY = 500`).

- **`settings.tsx`** — `definePluginSettings` with a toggle per action (`enableBan`, `enableKick`,
  `enableMute`, `enableDeafen`, …), `showToast`, `showNotification`, `notificationColor` (validated
  as hex), `history` and `voiceStates` (both `CUSTOM` type, auto-persisted), and `panel`
  (`COMPONENT` type).

- **`store.ts`** — Thin layer over the persisted state: `addAction`, `removeAction`,
  `clearHistory`, `getHistory`, `exportHistoryJson` (history) and `getVoiceState`, `setVoiceFlag`
  (per-user mute/deafen state). Enforces ordering and the size cap. **Contains no network calls** —
  everything is just data saved alongside the plugin's settings.

- **`utils.ts`** — `notifyAction` (toast + colored notification, with `noPersist: true` so the fake
  event doesn't pollute Discord's global log), `notifyVoiceOff` (toast when unchecking a checkbox),
  `formatDuration`, `describeAction`, `downloadHistory` (downloads the JSON via `saveFile`), and
  `normaliseColor`.

- **`components/ActionModal.tsx`** — Modal with a **reason** field and, for Timeout, a **duration**
  selector (presets + custom minutes). On confirm it calls `addAction` and `notifyAction`. Used only
  by moderation actions. **Never** calls the Discord API.

- **`components/SettingsPanel.tsx`** — Color picker (swatches + native picker), **Export JSON** and
  **Clear history** buttons (with a confirmation dialog), and the reactive history table with
  per-entry deletion.

- **`index.tsx`** — Patches the `user-context` menu: renders the red moderation items
  (via `Menu.MenuItem` with `color="danger"`) and the voice checkboxes (via `Menu.MenuCheckboxItem`),
  inserting them right after the native **Roles** area (with a fallback). Shows only what's enabled
  in settings.

### Security guarantees (client-side)

- The complete flow is `menu → modal/toggle → store (local data) → toast/notification`. There is no
  `fetch`, `RestAPI`, or any moderation endpoint involved.
- The Mute/Deafen checkboxes **don't** mute anyone — they only flip a field in a local object.
- The notification uses `noPersist: true` so the fake event isn't recorded in Discord's global
  notification log.
- The target is never touched — only the user's `id` and name are read to build the local record.

## Settings

Open **Discord → Settings → Equicord → Plugins → Fake Moderator**.

- **Enable/disable each action** — controls which entries appear in the menu (Ban, Kick, Timeout,
  Warn, Mute, and Deafen, individually).
- **Show toast** — shows (or not) the confirmation toast after each action.
- **Show notification** — shows (or not) the colored notification.
- **Notification color** — pick from preset swatches or a custom hex color.
- **Export JSON** — downloads the entire history as a `.json` file with a date/time in the name.
- **Clear history** — permanently deletes all local records (with confirmation).

### History

Each record stores: **target user** (name + ID), **action type**, **reason**, **duration** (when
applicable), and **date/time**. All of it is saved locally only, alongside the plugin's settings,
and never leaves your client.

## Installation

The plugin already lives in `src/userplugins/`, Equicord's official user-plugins directory.

```powershell
cd C:\Users\fofog\Documents\Equicord
pnpm install      # if you don't have the dependencies yet
pnpm build
pnpm inject       # injects into the Discord client — restart Discord afterwards
```

For development with automatic rebuilds, use `pnpm watch`. Then, under
**Discord → Settings → Equicord → Plugins**, enable **Fake Moderator**.
